# TERMINOLOGY.md

Этот документ задает единый словарь терминов курса.

Используй эти определения во всех материалах. Если нужно изменить определение, проверь все связанные документы.

## Правила работы с терминами

- Новый термин объясняется при первом использовании.
- Сначала дается простая аналогия, затем техническое объяснение.
- Один термин должен иметь одно базовое определение во всем курсе.
- Не используй английский термин без русского объяснения.
- Не заменяй термин случайными синонимами, если это может запутать студента.

## Базовые термины

| Термин | Простая аналогия | Рабочее определение |
| --- | --- | --- |
| ROS2 (Robot Operating System 2) | городская инфраструктура для программ робота | Среда, где отдельные программы робота обмениваются сообщениями и запускаются как система. |
| Node | отдельная программа | Выполняемый компонент ROS2, который решает одну задачу и общается с другими узлами. |
| ROS Graph (Robot Operating System Graph) | карта связей | Сеть узлов, топиков, сервисов и действий, существующих во время работы системы. |
| Workspace | рабочая директория | Папка, где лежат ROS2-пакеты и результаты сборки. |
| Package | небольшой проект | Минимальная единица организации кода, зависимостей и сборки в ROS2. |
| Topic | Telegram-канал | Именованный поток сообщений, куда publishers отправляют данные, а subscribers их читают. |
| Publisher | отправитель в канал | Объект в узле, который публикует сообщения в topic. |
| Subscriber | подписчик канала | Объект в узле, который получает сообщения из topic. |
| Message | письмо с полями | Тип данных, которым узлы обмениваются через topic. |
| Service | запрос-ответ | Механизм для короткой операции, где клиент отправляет запрос и ждет ответ. |
| Client | пользователь услуги | Узел или часть узла, которая отправляет service request. |
| Action | длительная задача | Механизм для задачи, у которой есть цель, прогресс, отмена и результат. |
| Parameter | настройка приложения | Именованное значение, которое меняет поведение узла без изменения кода. |
| Launch | сценарий запуска | Файл, который запускает несколько узлов и задает их параметры. |
| tf2 (Transform Library 2) | карта систем координат | Подсистема ROS2 для хранения и вычисления преобразований между координатными системами. |
| Frame | система координат | Именованная координатная система, привязанная к роботу, датчику или миру. |
| Transform | переход между координатами | Положение и ориентация одного frame относительно другого. |
| QoS (Quality of Service) | правила доставки | Набор настроек надежности, истории и долговечности сообщений. |
| DDS (Data Distribution Service) | служба доставки сообщений | Middleware, который передает данные между ROS2-узлами. |
| RMW (ROS Middleware) | адаптер к middleware | Слой ROS2, который связывает ROS2 API с конкретной реализацией DDS или другого middleware. |
| Discovery | автоматическое знакомство узлов | Процесс, при котором узлы находят друг друга в сети. |
| Executor | диспетчер выполнения | Компонент, который вызывает callbacks узла. |
| Callback | обработчик события | Функция, которую ROS2 вызывает при получении сообщения, таймере или ответе сервиса. |
| Lifecycle | жизненный цикл устройства | Модель управляемых состояний узла: создание, настройка, запуск, остановка и завершение. |
| colcon (Collective Construction) | сборщик проекта | Инструмент для сборки ROS2 workspace. |
| rviz2 (ROS Visualization 2) | визуализатор | Инструмент для просмотра данных робота: координат, сенсоров, карт и моделей. |
| rqt (ROS Qt) | панель инструментов | Набор GUI-инструментов для просмотра ROS Graph, topics, services и параметров. |
| URDF (Unified Robot Description Format) / Xacro (XML Macros) | технический паспорт тела робота | Формат описания links, joints, геометрии, масс и связей робота для ROS2-инструментов и симуляции. |
| Gazebo/Ignition | испытательный полигон | Симулятор, где проверяют модель робота, сенсоры и управление до физического запуска. |
| ros2_control | коробка передач между ROS2 и приводами | Слой контроллеров и hardware interfaces, который связывает команды ROS2 с реальными или симулируемыми приводами. |
| Nav2 (Navigation 2) | навигатор робота | Стек ROS2 для построения маршрута и движения мобильного робота к цели по карте. |
| SLAM (Simultaneous Localization And Mapping) | составление карты на ходу | Одновременное построение карты и оценка положения робота на этой карте. |
| robot_localization | фильтр положения | Набор ROS2-узлов для объединения одометрии, IMU и других измерений в оценку положения робота. |
| MoveIt2 | планировщик движений руки | Стек ROS2 для планирования движений манипулятора с учетом модели, ограничений суставов и столкновений. |
| YOLO (You Only Look Once) | быстрый детектор объектов | Модель компьютерного зрения, которая находит объекты на изображении и выдает классы, координаты и confidence. |
| LLM (Large Language Model) bridge | переводчик команд человека | Узел или набор узлов, который переводит текстовую или голосовую команду в разрешенные high-level actions робота. |
| Behavior Tree | дерево поведения | Способ описать выполнение задачи как набор проверок, действий и fallback-веток. |
| Safety layer | слой безопасности | Узлы и правила, которые ограничивают опасные действия, контролируют E-stop, watchdog и запреты на прямое управление приводами. |

## Допустимые короткие формулировки

Используй эти формулировки в слайдах и вводных абзацах:

- ROS2 - среда для связи программ робота.
- Node - отдельная программа в ROS2.
- Topic - поток сообщений от отправителей к подписчикам.
- Service - короткий запрос и ответ.
- Action - длительная задача с прогрессом и результатом.
- Launch - сценарий запуска нескольких узлов.
- tf2 - дерево координат робота.
- QoS - настройки доставки сообщений.
- Workspace - папка, где собирается ROS2-проект.
- Package - единица кода и зависимостей.
- URDF/Xacro - описание тела робота для ROS2 и симуляции.
- ros2_control - стандартный слой между ROS2 и приводами.
- Nav2 - навигационный стек мобильного робота.
- MoveIt2 - стек планирования движений манипулятора.
- LLM bridge - мост от команды человека к разрешенному действию робота.

## Нежелательные формулировки

Не начинай объяснение с таких фраз:

- "ROS2 представляет собой современную распределенную программную платформу".
- "DDS является спецификацией middleware для data-centric publish-subscribe".
- "Executor осуществляет orchestration callback groups".

Сначала дай практическое объяснение, затем добавляй точность.

## Правило уточнения

Если простая аналогия технически неточна, сразу после нее добавь ограничение.

Пример:

> Topic похож на Telegram-канал. Но в ROS2 сообщения обычно имеют строгий тип, например координаты или скорость.
