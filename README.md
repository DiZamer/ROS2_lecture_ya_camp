# Курс по ROS2

Образовательный комплект для лекции на 80 минут по ROS2 для студентов магистратуры физико-технического университета.

**Главная цель**: через 80 минут студент перестает бояться ROS2 и может самостоятельно разработать небольшое распределенное робототехническое приложение.

Лекция входит в программу интенсивного воркшопа по робототехнике. После нее идут сенсорика, цифровые двойники, YOLO, SLAM, Nav2, VLA, манипуляторы, sim-to-real, M2M, информационная безопасность и финальная интеграция. **ROS2 — общий инженерный язык для всех следующих тем**.

## Как устроен курс

Материал организован в три уровня:

| Уровень | Что это | Где находится | Для кого |
| --- | --- | --- | --- |
| **1. Лекция** | 80 минут (40+40): объяснение концепций, схемы, фрагменты кода, CLI-демонстрации | `1_lecture/`, `1_slides/`, `1_demo/` | Все студенты на лекции |
| **2. Практика** | Самостоятельные материалы: статьи, практики, код, шпаргалки — «сделай так и заработает» | `2_knowledge/`, `2_practice/`, `2_code/`, `2_cheatsheets/` | Студент после лекции |
| **3. Робот** | Архитектурный кейс мобильного манипулятора TIAGo — как те же механизмы работают в реальной системе | `3_Robot/TIAgo_humble/` | Углубленное изучение |

Каждый блок лекции содержит две ссылки:
- **Уровень 2**: подробный материал и практику для самостоятельного повторения
- **Уровень 3**: где эта технология используется в архитектуре робота

## Среда выполнения

**ROS2 не устанавливается на хост.** Все работает в контейнерах:

- **Уровень 2** — общий Dev Container в `.devcontainer/` (ROS2 Jazzy, Ubuntu 24.04)
- **Уровень 3** — отдельный контейнер в `3_Robot/TIAgo_humble/` (ROS2 Humble, Ubuntu 22.04, Gazebo Classic 11, Nav2, MoveIt2)

## Быстрый старт

```bash
# 1. Открыть проект в VS Code с Dev Containers
# 2. Пересобрать контейнер при первом запуске
# 3. Проверить окружение
ros2 --help
ros2 run demo_nodes_cpp talker
```

## Маршрут студента

1. **На лекции**: понять архитектуру ROS2, увидеть работающий код, запомнить схему выбора topic/service/action
2. **После лекции**: открыть `2_knowledge/ros_architecture.md` → создать workspace через `2_practice/01_workspace.md` → написать первый узел через `2_practice/02_package.md`
3. **Для углубления**: изучить `3_Robot/TIAgo_humble/` — как учебные примеры масштабируются до архитектуры мобильного манипулятора

## Целевая аудитория

Студенты магистратуры по робототехнике, мехатронике, автономным мобильным роботам и интеллектуальным робототехническим системам.

**Предполагается, что студент знает**: C++, Python (базово), Linux, устройство компьютера, основы робототехники.

**Не предполагается**, что студент знает ROS2, DDS, middleware, ROS Graph, Executor, QoS, tf2, launch, colcon или устройство ROS2 workspace.

## Главные принципы курса

- Практика важнее теории
- Каждый термин объясняется при первом использовании
- Сложная концепция сначала через аналогию, затем технически
- Каждая тема завершается практическим результатом
- Код и команды работают при копировании без изменений
- Все нетривиальные утверждения имеют ссылку на источник
